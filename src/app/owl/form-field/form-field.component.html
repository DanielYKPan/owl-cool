Import

Label

Label in owl-form-field is a caption on top of the form field control when the control does not contain any value. When the form field control has value, the label floats above it by default. The label for a form field can be defined either by wrapping content inside owl-label, by using owlLabel directive on your html element, by setting the placeholder attribute on the form control or using the owl-placeholder component. If there are multiple values (such as there are both label and placeholder values), the owlLabel will take precedence.

The placeholder specifies a short hint that describes the expected value of form control field (e.g. a sample value or a short description of the expected format). It can be specified using the placeholder property on the form control field, or by wrapping html content with owl-placeholder and using it inside the form control field.

[floatLabel] can be set on owl-form-field to change the behavior of floating the label. There are the options:

  • [floatLabel]=" 'auto' " -- the label would float as the form control field has value. (default)
  • [floatLabel]=" 'always' " -- the label would be always above the form control field.
  • [floatLabel]=" 'never' " -- the label would never float, and when the form control field has value, the label would be disappear.

If the form field control is marked with [required]="true", an asterisk symbol(*) will be appended to the label to indicate the fact that it is a required field. If this is a undesired effect, this can be disabled by setting the [hideRequiredMarker] to true on owl-form-field.

Error Message

Form field error messages can be show under the form field underline by using owl-error inside owl-form-field. Errors messages are hidden initially and will appear on invalid form fields after user has interacted with the element or the parent form has been submitted. By design, owl-form-field has enough space to display only one error message at a time. If a form field has more than one error messages, user can use ngIf or ngSwitch to choose which error message should be displayed. You could learn more this from here.

Hints Message

Hint message are descriptive text that is shown below the form field’s underline. Each owl-form-field can have up to two hint messages: one on the start-align of the line, and one on the end. Hint message can be created either by wrapping html content or text content with owl-hint or by setting property [hintLabel] in owl-form-field. By default, hint message will be set as the start-align hint. Set property [align] to ‘start’ or ‘end’ to specify the place of the hint message. When there is an error message shown in the form field, the hint messages would be hidden.

Prefix & Suffix

Content can be included before and after the input tag, as a prefix or suffix. Adding the owlPrefix directive to an element inside the owl-form-field will designate it as the prefix. Similarly, adding owlSuffix will designate it as the suffix.

Properties of owl-form-field

Name Type Optional Default Description
floatLabel 'always' | 'never' | 'auto' Optional 'auto' Whether the label should always float, never float or float as the user types.
'auto' -- the label would float as the user types
'always' -- the label would always float
'never' -- the label would never float
hideRequiredMarker boolean Optional false Whether the required marker should be hidden.
hintLabel string Optional null Defines the text for the form field hint message. If the owl-hint was used, this hintLabel content would be shown.

Properties of owl-hint

Name Type Optional Default Description
align 'start' | 'end' Optional 'start' Whether to align the hint message at the start or end of the underline.
id string Optional owl-hint-0 Unique ID for the hint. Used for the aria-describedby on the form field control.

Properties of owl-error

Name Type Optional Default Description
id string Optional owl-error-0 Unique ID for the error message.
