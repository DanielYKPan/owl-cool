Owl Input is an Angular directive for native HTML ]]> and ]]>. It works inside of an ]]> , having floating placeholder and underline animation.

Getting Start

All of the attributes that can be used with native HTML ]]> and ]]> can be used on elements inside ]]> .

The following input types can be used with owlInput: date, datetime-local, email, month, number, password, search, tel, text, time, url and week.

Use with @angular/forms

]]> is compatible with @angular/forms and supports both FormsModule and ReactiveFormsModule. This means Angular Form directives ngModel and formControl can be used to bind to your input value.

Owl Form Field Features

There are a number of owl-form-field features that can be used with ]]> or ]]>. These include error messages, hint text, prefix & suffix. For additional information about these features, see the form field documentation.

Placeholder

The placeholder specifies a short hint that describes the expected value of an input field (e.g. a sample value or a short description of the expected format). This short hint is displayed in the input field before the user enters a value. When text value is present, the indicative text will float above this input zone.

User could hide the placeholder by setting property [floatPlaceholder] to 'never', or have a always floated placeholder by setting [floatPlaceholder] to 'always'.

Customise Error Matcher

The owl-form-field allows you to associate error messages with your owlInput. By default, these error messages are shown when the control is invalid and either the user has interacted with (touched) the element or the parent form has been submitted. If you wish to override this behavior (e.g. to show the error as soon as the invalid control is dirty or when a parent form group is invalid), you can use the errorStateMatcher property of the owlInput. The property takes an instance of an ErrorStateMatcher object. An ErrorStateMatcher must implement a single method isErrorState which takes the FormControl for this owlInput as well as the parent form and returns a boolean indicating whether errors should be shown. (true indicating that they should be shown, and false indicating that they should not.)
