Description

Owl Notifier is an Angular service for providing notifications. It is a well designed, fully animated, highly customizable, and a easy-to-use notification service for your Angular application.

Import

Getting Start

Open a Notifier

Owl Notifier can contain either a string message or a custom component as notifier's content.

In either case, a OwlNotifierRef is returned. This can be used to dismiss the notifier or to receive notification of when the notifier is dismissed. For simple messages with an action, the OwlNotifierRef exposes an observable for when the action is triggered. If you want to close a custom notifier that was opened via openViaComponent, from within the component itself, you can inject the OwlNotifierRef.

Dismiss a Notifier

A notifier can be dismissed manually by calling the dismiss() method on the OwlNotifierRef returned from the call to open() or openViaComponent().

By default, only one notifier can be shown at a time. If a new notifier is opened while the previous one is still showing, the previous one would be automatically dismissed. If you want to have more than one notifiers that could be showing at the same time, you could set maxStack in Global Notifier Config.

In addition, each notifier could also be given a life via the optional configuration object. Notifiers with a life would be automatically dismiss after the certain milliseconds. The default life of notifier is 0, and it would not dismiss automatically.

Position

By default, notifier would be shown in the bottom-center of the window. This can be configured via Global Notifier Config or the optional configuration object. ValidverticalPosition setting is 'top' and 'bottom', and valid horizontalPosition setting is 'left', 'right', 'center'.

Global Notifier Config

Default notifier config options can be specified by providing an instance for OWL_NOTIFIER_CONFIG in your application’s root module.

Share Data in custom notifier

You can share data with the custom notifier, that you opened via the openViaComponent method, by passing it through the data property.

To access the data in your custom component, you have to use the OWL_NOTIFIER_DATA injection token:

Options In OwlNotifierConfig

Name Type Default Description
announcementMessage string null Message to be announced by the AriaLiveAnnouncer
data any null Data to be injected into the custom notifier component.
clickToClose boolean false When specify to true, the notifier would be dismissed when is clicked.
id string null ID for the notifier. If omitted, a unique one will be generated.
life number 0 The length of time in milliseconds to wait before automatically dismissing the notifier. Default is 0, and the notifier would not be automatically dismissed.
maxStack number 1 Defines maximum number of notifiers can be shown on the page at the same time
notifierClass string | string[] null The extra class or classes for the notifier
type 'question' | 'warning' | 'info' | 'success' | 'error' null Defines the type of the notifier. Each type would have a corresponding icon and theme color.
verticalPosition 'top' | 'bottom' 'bottom' Defines the vertical position of the notifier
horizontalPosition 'left' | 'center' | 'right' 'center' Defines the horizontal position of the notifier
viewContainerRef ViewContainerRef null Defines the view container to place the overlay for the notifier into.
