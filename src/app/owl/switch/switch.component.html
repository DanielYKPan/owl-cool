Description

Owl Switch is an Angular component to allow user to toggle boolean value. It is an on/off control that can be toggled via clicking or dragging. It is well designed with styles and animations.

Note: the sliding behavior for this component requires that HammerJS is loaded on the page. You could learn more about this from here.

Import

Getting Start

Owl Switch is set up for both Template-driven Forms and Reactive Forms. User can manipulate the switch's value using any of the form directives from the core FormsModule or ReactiveFormsModule: ngModel, formControl, etc.

Switch Label

The default position of the label is after the switch. User could change property [labelPosition] to 'before'. If user doesn't want the label to appear next to the switch, he can use aria-label or aria-labelledby to specify an appropriate label.

Properties for owl-switch

Name Type Optional Default Description
aria-label string Optional null Attached to the aria-label attribute of the switch
aria-labelledby string Optional null Users can specify the 'aria-labelledby' attribute which will be forwarded to the switch
checked boolean Optional false Whether the switch is checked.
disabled boolean Optional false When specified to true, the switch would be disabled.
disableRipple boolean optional false When specified to true, the ripple effect would be disabled.
id string Optional 'owl-switch-0' A unique id for the switch input. If none is supplied, it will be auto-generated.
labelPosition 'after' | 'before' Optional 'after' Whether the label should appear after or before the switch.
name string Optional null The name attribute of the switch
required boolean Optional false Whether the switch's value is required.
tabIndex number Optional null Index of the element in tabbing order
vertical boolean Optional false Defines whether the switch to be vertical
