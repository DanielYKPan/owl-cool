To use OwlNG, User has to have a Angular 5+ app.

For help to start with an Angular 5+ app, check out the Angular CLI.

For existing Angular 5+ apps, follow these steps to begin using OwlNG.

Step 1: Install OwlNG

Step 2: Animations

Some OwlNG components use Angular Animations to improve the user experience and deal with complex transitions. Therefore, user needs to install @angular/animations and import BrowserAnimationsModule to their application module.

Note: @angular/animations are built on top of the standard WebAnimation API and run natively on browsers that support it. For other browsers, a polyfill is required. You could learn more about this from here.

If you prefer to disable animation effect, use NoopAnimationsModule instead.

Step 3: Import OwlNG component modules

Import the module that you want to use. For example: use the OwlRadioModule

Step 4: Include the styles

Including the OwlNG styles is required. If you are using Angular CLI, you can add this to your styles.css:

If you are not using the Angular CLI, you can include the theme.min.css via a link element in your index.html.

Step 5: Gesture Config

Some of the OwlNG components (OwlSliderModule, OwlTooltipModule, OwlSwitchModule) rely on HammerJS for gestures. In order to get the full feature and user experience of the OwlNG components, HammerJS must be loaded into the application.

You can add HammerJS to your application via NPM, a CDN (such as the Google CDN), or served directly from your app.

After installing, import it on your app's entry point (e.g. src/main.ts).

If you use OwlNG Components on a lazy-loaded module, you also have to add custom gesture config in your AppModule.
