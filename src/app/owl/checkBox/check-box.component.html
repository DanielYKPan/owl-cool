Description

Owl Checkbox owl-check-box is an Angular component. It provides the same functionality as a native HTML Input Element ]]>, enhanced with styles and animation.

Import

Getting Start

Owl Checkbox can either be used in multiple selection with other checkboxes or as a single checkbox to provide a boolean value. It is set up for both Template-driven Forms and Reactive Forms. User can manipulate the select's value using any of the form directives from the core FormsModule or ReactiveFormsModule: ngModel, formControl, etc.

Boolean Value

By default, Owl Checkbox is in binary mode. Users could use Angular built-in ngModel to bind checkbox to a boolean variable.

Indeterminate Checkbox

Sometimes, a checkbox could be in an indeterminate state. You could control the checkbox indeterminate value via property [(indeterminate)]. While the indeterminate property of the checkbox is true, it will render as indeterminate regardless of the checked value. Any interaction with the checkbox by a user (i.e., clicking) will remove the indeterminate state.

Checkbox Label

The checkbox label is provided as the content of your checkbox. The label can be positioned before or after the checkbox by setting the property [labelPosition] to 'before'(default) or 'after'.

If you don't want the label to appear next to the checkbox, you can use [aria-label] or [aria-labelledby] to specify an appropriate label.

Appearance

    public toggleMeatCheck( event: any ): void {
        if (event.checked) {
            this.selectedGroceries = this.selectedGroceries.filter(( v ) => {
                return v !== 'Pork' && v !== 'Beef';
            });
            this.selectedGroceries = [...this.selectedGroceries, 'Pork', 'Beef'];
        } else {
            this.selectedGroceries = this.selectedGroceries.filter(( v ) => {
                return v !== 'Pork' && v !== 'Beef';
            });
        }
    }

    public onMeatChange( event: any ): void {
        const hasPork = this.selectedGroceries.includes('Pork');
