Description

Owl DateTimePicker is an Angular component that allows users to enter a date and/or time either through text input, or by choosing a date from the calendar and/or a time from the timer. It is made up of several components and directives that work together.

Note: Currently, OwlDateTimePicker is not part of OwlNG package. Users have to install this package via npm or yarn.

Getting Start

1. Install via npm or yarn

2. Add styles. If you are using Angular CLI, you can add this to your styles.css

If you are not using the Angular CLI, you can include the picker.min.css via a-link HTML element in your index.html.

3. Add OwlDateTimeModule and OwlNativeDateTimeModule to your @NgModule like example below

4. Connecting a picker to an input and a trigger

Select Mode

By default, OwlDateTimePicker allows users to select a single DateTime Object. User could select range date time by setting property [selectMode] to 'range', 'rangeFrom' or 'rangeTo'. 



]]>
  • 'range' -- the input would show both the 'from' and 'to' value, and the picker could select both 'from' and 'to' value and switch between them.
  • 'rangeFrom' -- the input would show only the 'from' value, and the picker could only change the 'from' value.
  • 'rangeTo' -- the input would show only the 'to' value, and the picker could only change the 'to' value.

Setting the calendar starting View

The [startView] property of owl-date-time can be used to set the view that will show up when the calendar first opens. It can be set to month, year, or multi-years; by default it will open to month view. In multi-years view the user will see a range of years, and in year view user will see all the months of the year. Picker would proceed to month view after choosing a month of a year. 




// calendar open on 'year' view


]]>

The moment that the picker opens to is determined by first checking if any date or time is currently selected, if so it will open to the month or year containing that date and timer is set to that moment. Otherwise it will open to the month or year containing today's date and set to current time. This behavior can be overridden by using the [startAt] property of owl-date-time. In this case the calendar will open to the month or year containing the startAt date and timer is set to startAt time. 



]]>

Date Time Validation

There are three properties that add date-time validation to the picker's input. The first two are the [min] and [max] properties. In addition to enforcing validation on the input, these properties will disable all dates on the calendar or time value on the timer before or after the min and max values and prevent the user from advancing the calendar past the month or year (depending on current view) containing the min or max date. 



]]>

The second way to add date-time validation is using the owlDateTimeFilter property of the picker's input. This property accepts a function of D => boolean (where D is the date type used by the picker). A result of true indicates that the date-time is valid and a result of false indicates that it is not. Again this will also disable the dates on the calendar that are invalid and prevent time being set on the timer.

Validation Error

There are five validation errors that can be checked:

  • A value that violates the min property will have a owlDateTimeMin error.
  • A value that violates the max property will have a owlDateTimeMax error.
  • A value that violates the owlDateTimeFilter property will have a owlDateTimeFilter error.
  • An invalid date-time value that input has will have a owlDateTimeParse error.
  • If the 'from' and 'to' values in range selection are invalid ('from' value after 'to', or 'to' value before 'from'), it will have a owlDateTimeRange error.

Choosing a date-time implementation and date-time format settings

OwlDateTime picker can work with a variety of different date-time implementations. Users must either import one of the pre-made modules(OwlNativeDateTimeModule, OwlMomentDateTimeModule) or built a customized date-time module. There are two pre-made modules:

  • OwlNativeDateTimeModule-- support for native JavaScript Date object
  • OwlMomentDateTimeModule-- support for MomentJS object. (need to npm install ng-pick-datetime-moment and moment packages)

Build you own custom DateTimeAdapter

If neither one the pre-made modules is suitable for your project, you could customize your own date-time implementation. Making the picker work with a different date-tim implementation is as easy as extending DateTimeAdapter, and using your subclass as the provider. You will also want to make sure that the OWL_DATE_TIME_FORMATS provided in your app are formats that can be understood by your date-time implementation.

Localizing labels and messages

The various text strings used by the picker are provided through OwlDateTimeIntl. Localization of these messages can be done by providing a subclass with translated values in your application root module.

Accessibility

Here are the keyboards shortcuts to handle the picker:

  • ALT + DOWN_ARROW -- Open the picker
  • ESCAPE -- Close the picker

In month view:

  • LEFT_ARROW -- Go to previous day
  • RIGHT_ARROW -- Go to next day
  • UP_ARROW -- Go to same day in the previous week
  • DOWN_ARROW -- Go to same day in the next week
  • HOME -- Go to the first day of the month
  • END -- Go to the last day of the month
  • PAGE_UP -- Go to previous month
  • ALT + PAGE_UP -- Go to previous year
  • PAGE_DOWN -- Go to next month
  • ALT + PAGE_DOWN -- Go to next year
  • ENTER -- Select current date

In year view:

  • LEFT_ARROW -- Go to previous month
  • RIGHT_ARROW -- Go to next month
  • UP_ARROW -- Go to previous 3 months
  • DOWN_ARROW -- Go to next 3 months
  • HOME -- Go to the first month of the year
  • END -- Go to the last month of the year
  • PAGE_UP -- Go to previous year
  • ALT + PAGE_UP -- Go to previous 10 years
  • PAGE_DOWN -- Go to next year
  • ALT + PAGE_DOWN -- Go to next 10 years
  • ENTER -- Select current month

In multi-years view:

  • LEFT_ARROW -- Go to previous year
  • RIGHT_ARROW -- Go to next year
  • UP_ARROW -- Go up a row (back 3 years)
  • DOWN_ARROW -- Go down a row (forward 3 years)
  • HOME -- Go to the first year in the current range
  • END -- Go to the last year in the current range
  • ENTER -- Select current year

Properties for owl-date-time

Name Type Optional Default Description
pickerType 'both' | 'calendar' | 'timer' optional 'both' Set the type of the dateTime picker. 'both': show both calendar and timer; 'calendar': only show calendar; 'timer': only show timer.
pickerMode 'popup' | 'dialog' optional 'popup' Specify the style the picker would open as.
startView 'month' | 'year' | 'multi-years' optional 'month' Specify the view that the calendar should start in.
startAt T/null optional null Specify the picker initial moment
firstDayOfWeek number optional 0 Set the first day of week. Valid value is from 0 to 6. 0: Sunday ~ 6: Saturday
showSecondsTimer boolean optional false When specify it to true, it would show a timer to configure the second's value. Beware, this dose not guarantee the second's value shows in the input. User also need to set appropriate format to show the second's value.
hideOtherMonths boolean optional false Whether to hide dates in other months at the start or end of the current month
hour12Timer boolean optional false When specify it to true, the timer would be in hour12 format mode
stepHour number optional 1 Hours to change per step.
stepMinute number optional 1 Minutes to change per step.
stepSecond number optional 1 Seconds to change per step.
disabled boolean optional false When specify to true, it would disable the picker.
backdropClass string | string[] optional null Custom class for the picker backdrop.
panelClass string | string[] optional null Custom class for the picker overlay panel.

Name Parameters Description
afterPickerOpen null Event emitted when this picker is opened.
afterPickerClosed null Event emitted when this picker is closed.
yearSelected T Event emitted when year is selected. This doesn't imply a change on the selected date.
monthSelected T Event emitted when month is selected. This doesn't imply a change on the selected date.
